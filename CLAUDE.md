## Project Overview

`flutter_boilerplate` is a Flutter application scaffold generated by `flutter create` (project type `app`), currently at the default state: the counter demo in `lib/main.dart`. It is intended as a **starting point** — expect new sessions to add real structure (routing, state management, feature folders) on top of the bare scaffold.

Targets all six Flutter platforms: `android/`, `ios/`, `linux/`, `macos/`, `web/`, `windows/`.

## Dart 3.12 dot-shorthand syntax

`lib/main.dart` uses Dart 3.12 **dot-shorthand** syntax, where the type before a `.member` is inferred from context. This is intentional and valid — not a typo or missing identifier:

- `colorScheme: .fromSeed(seedColor: ...)` → resolves to `ColorScheme.fromSeed(...)`
- `mainAxisAlignment: .center` → resolves to `MainAxisAlignment.center`

Preserve this style when editing existing code; the project SDK supports it. Older analyzers/tooling that predate Dart 3.12 will flag these as errors, so ensure any tooling matches the pinned SDK.

## Architecture

The app is a single-file entry point (`lib/main.dart`) with three widgets: `MyApp` (root `MaterialApp` + theme), `MyHomePage` (stateful shell), and `_MyHomePageState` (counter state via `setState`). There is no routing, dependency injection, or state-management layer yet — add these deliberately as the app grows rather than assuming they exist.

`test/widget_test.dart` imports the app via `package:flutter_boilerplate/main.dart`; the package name comes from `pubspec.yaml` (`name: flutter_boilerplate`), so any import of first-party code uses that `package:flutter_boilerplate/...` prefix.
